# 제품 요구사항 정의서 (PRD): AI 시민제안 Co-Pilot (v1.1)

- **문서 버전:** 1.1 (한글 서식 파일 연동)
- **작성일:** 2025년 9월 2일
- **작성자:** 경영평가 마스터 (AI 컨설턴트)
- **프로젝트명:** AI 시민제안 Co-Pilot 서비스 프로토타입 개발

---

## 1. 문서 개요

본 문서는 김포도시관리공사(이하 '공사')의 'AI 시민제안 Co-Pilot' 서비스 개발 요구사항을 정의합니다. 본 서비스는 시민이 아이디어의 핵심만 입력하면, AI가 **미리 준비된 공사 공식 한글 서식 파일('시민제안서식.hwp')의 지정된 위치('누름틀')에** 제안서 초안을 자동으로 작성하고, 최종적으로 완성된 한글(HWP) 파일을 시민에게 제공하는 것을 목표로 합니다.

---

## 2. 서비스 목표 및 비전

- **비전:** 디지털 포용 행정을 구현하고, 시민과 함께 정책을 만들어가는 혁신적인 참여 모델 선도
- **핵심 목표:**
    1.  **참여 장벽 완화:** 시민 제안의 심리적·기술적 장벽 50% 완화
    2.  **제안 품질 향상:** 양질의 제안 접수를 통한 정책 채택률 10%p 이상 향상
    3.  **행정 효율 증대:** 정형화된 제안서 접수를 통한 담당자 검토 및 처리 기간 20% 단축

---

## 3. 사용자 스토리 (User Stories)

1.  **(문서 작성이 어려운 시민):** "나는 좋은 아이디어가 있지만, 공공기관에 내는 서류는 어떻게 써야 할지 막막하다. AI가 내 아이디어를 논리적인 제안서로 만들어주면 자신감을 갖고 제출할 수 있을 것이다."
2.  **(바쁜 직장인):** "나는 평소에 떠오르는 아이디어를 메모해두지만, 이걸 정식 제안서로 만들 시간이 없다. AI가 내 메모를 바탕으로 빠르게 제안서 초안을 만들어준다면, 바쁜 와중에도 시정에 참여할 수 있을 것이다."
3.  **(제안서 검토 담당자):** "나는 하루에도 수많은 제안서를 검토하는데, 내용이 중구난방이라 핵심을 파악하기 어렵다. AI가 논리적인 서식에 맞춰 제안서를 만들어준다면, 훨씬 빠르고 효율적으로 검토 업무를 처리할 수 있을 것이다."

## 4. 핵심 기능 명세 (Feature Specifications)

### 4.1. 프런트엔드 (시민 이용 화면)

- **F-01: 가이드 기반 아이디어 입력 양식**
    - 사용자는 4개의 명확한 질문(`문제점`, `수혜자`, `해결방안`, `기대효과`)에 대한 답변을 각각의 텍스트 입력창에 입력한다.
- **F-02: AI 초안 생성 요청 버튼**
    - 사용자는 'AI로 제안서 초안 만들기' 버튼을 클릭하여 백엔드에 제안서 생성을 요청한다.
- **F-03: 블록형 결과 확인 및 편집 화면**
    - AI가 생성한 결과는 공사 공식 서식의 항목(`제안명`, `현황 및 문제점`, `개선방안`, `기대효과`)에 맞춰 각각의 '블록'으로 구분되어 표시된다.
    - 사용자는 각 블록의 내용을 직접 수정할 수 있다.
- **F-04: 한글(HWP) 파일 다운로드 버튼**
    - 사용자는 최종 확인된 내용을 '한글(HWP) 파일로 다운로드' 버튼을 클릭하여 자신의 PC에 저장할 수 있다.

### 4.2. 백엔드 (서버 로직)

- **B-01: 제안서 내용 생성 기능**
    - 프런트엔드에서 받은 4가지 핵심 아이디어 텍스트를 입력값으로 받는다.
    - 사전에 정의된 시스템 프롬프트(Prompt)와 사용자 입력값을 조합하여 Google Gemini API에 요청한다.
    - API 응답(생성된 텍스트)을 `제안명`, `현황 및 문제점`, `개선방안`, `기대효과` 4가지 항목으로 구조화하여 파싱(Parsing)한다.
- **B-02: 개인정보 처리 배제**
    - AI는 제안자의 개인정보(성명, 주소, 연락처)를 **절대 수집하거나 처리하지 않는다.** 오직 제안 내용 생성에만 관여한다.
- **B-03: 한글(HWP) 파일 생성 및 다운로드 기능 (핵심)**
    - **(1) 서식 파일 로드:** 사용자의 다운로드 요청 시, 서버에 미리 저장된 **`시민제안서식.hwp` 파일을 불러온다.**
    - **(2) 누름틀에 내용 삽입:** B-01에서 생성된 텍스트를 한글 서식 파일 내의 각 **'누름틀' 이름(예: `{{제안명}}`, `{{문제점}}`)에 정확히 찾아가 내용을 삽입**한다.
    - **(3) 파일 생성 및 제공:** 내용이 모두 채워진 한글 파일을 고유한 파일명(예: `시민제안서_결과.hwp`)으로 생성하여 사용자에게 다운로드 스트림으로 제공한다.

## 5. 시스템 요구사항

- **프런트엔드:** 표준 HTML5, CSS3, JavaScript를 준수해야 하며, 웹 접근성 지침을 고려해야 한다.
- **백엔드:** Python(Flask 프레임워크 권장)으로 구현한다.
- **AI 모델:** Google Gemini Pro API 또는 동급 성능의 LLM API를 사용한다.
- **HWP 생성:**
    - 백엔드 서버는 **Windows Server 운영체제**이어야 한다.
    - 백엔드 서버에 **한글과컴퓨터 오피스 소프트웨어**가 설치되어 있어야 한다.
    - Python 라이브러리 **`pywin32`** 를 사용하여 한글 파일을 제어한다.
    - **`시민제안서식.hwp` 파일**이 서버 내 지정된 경로에 위치해야 한다.
